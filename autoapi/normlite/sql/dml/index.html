class normlite.sql.dml.Insert[source]

Bases: normlite.sql.base.Executable

Provide the interface for all executable SQL statements.

This base class implements the command design pattern for execution of clause elements. The execute() method has a base implementation of the execution flow that can be used as default implementation in the subclasses.

It provides a post execution hook with the _post_exec() that is intended to be optionally implemented in the subclasses.

Added in version 0.7.0: This bas class fully supports the connection-driven execution flow of SQL statements.

__visit_name__ = 'insert'

Provide the SQL INSERT node to create a new row in the specified table.

This class provide a generative implementation of the SQL INSERT node to be executed on a normlite.connection.Connection.

Usage:
>>> # create a new insert statement for the table students
>>> stmt = Insert(students)
...
>>> # create a new insert statement and specify the ``RETURNING`` clause.
>>> stmt = Insert(students).returning(students.c.id, students.c.name)
...
>>> # specify the values to be inserted as keyword arguments
>>> stmt.values(id=123456, name='Isaac Newton', grade='B')
...
>>> # specify the values to be inserted as dictionary
>>> stmt.values({'id': 123456, 'name': 'Isaac Newton', 'grade': 'B'})
...
>>> # specify the values to be inserted as tuple
>>> stmt.values((123456, 'Isaac Newton', 'B'))

Important

The Insert has always by default the Notion specific columns as RETURNING clause. That is, the normlite.CursorResult methods of an insert statement always returns rows with the columns corresponding to the Notion specific ones. You specify the Insert.returning() to get additional columns in the rows returned by the normlite.CursorResult methods.

Note

The Insert can also be constructed without specifying the values. In this case, the parameters passed in the normlite.connection.Connection.execute() are bound as VALUES clause parameters at execution time.

Changed in version 0.7.0: The old construct has been completely redesigned and refactored. Now, the new class provides all features of the SQL INSERT statement.
